# はじめに
## 目標
- `Nvim`でzennレポジトリを開く
- `npx zenn preview`を実行
- 画面左にエディタ(nvim)、右にChromeプレビューを表示

をコマンド一発で行いたい
RaycastのCustom Windowが一番簡単だろうけど、課金したくないので別の方法を探す

## 必要なもの

- [Raycast script command](https://www.raycast.com/): コマンド実行 
- [yabai](https://github.com/koekeishiya/yabai) : ウィンドウマネージャ
    インストール↓
    ```bash
    $ brew install koekeishiya/formulae/yabai
    ```
- 各人のターミナル (この記事は[Ghostty](https://ghostty.org/)を使っています)

# 手順
(おおまかに) やりたいこと
1. NvimでZennレポジトリを開く
2. `npx zenn preview`を実行
3. Chromeで`localhost:8000` を開く
4. 左右に配置する

## script commandの作成

raycastの設定で色々やる

## 実行ファイルを作る

コード完成形

以下解説

### 1,2.  NvimでZenn開いて`npx run preview` 

```sh
/Applications/Ghostty.app/Contents/MacOS/ghostty -e zsh -lc "
    cd ~/src/github.com/nematatu/zenn &&
    npx zenn preview > /dev/null 2>&1 &
    nvim" > /dev/null 2>&1 & disown
```

- `/Applications/Ghostty.app/Contents/MacOS/ghostty -e zsh` : Ghosttyでzshを起動
- `-l` : ログインシェルで起動 (`.zshrc` 読み込んでくれる)
- `-c` : コマンド実行

実行コマンド

```bash
cd <path/to/zenn> &&
npx zenn preview > /dev/null 2>&1 &
nvim
```

やってること:
zennディレクトリに移動して、バックグラウンドでサーバーを立ててnvimを開く

:::message

`&` と`&&` の違い

```
com1 && com2
```

- `&&`は<com1>が終了したあと<com2>を実行したいときに使う
- `&` はバックグラウンドで実行させたいときに使う

  (今回の場合、Nvimを一番前に出したいのでサーバー起動は裏に回ってもらった)

:::
- `& disown` : プロセスを切り離す

  (これがないと、Nvimプロセスを終了するまでRaycastが使用不可になってしまうので付けた)
- `sleep 1` : サーバー起動までちょっと待つ

### 3. Chromeで`localhost:8000`を開く

```bash
osascript <<'APPLESCRIPT'
tell application "Google Chrome"
    activate
    set newWindow to make new window
    tell newWindow
        set URL of active tab to "http://localhost:8000"
    end tell
end tell
APPLESCRIPT
```

やってること:
zennをプレビューする専用のChromeを立ち上げます

ここまでで、nvimとChromeがコマンド一発で起動が出来ました
